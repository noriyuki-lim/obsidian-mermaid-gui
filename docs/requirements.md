# Mermaid GUI Editor 要件定義

## 1. 背景と課題
- Mermaidはテキストで図を書ける手軽さが強みだが、ノード数が増えると記法の編集・レイアウト調整・リファクタがつらい。
- 既存資産（README・設計メモに埋め込まれた `mermaid` ブロック）を視覚的に編集する手段が乏しい。
- 一方、PowerPoint / Visio / draw.io はマウスで作図できるが、出力がバイナリ／独自XMLでGit差分が読めず、Markdownに埋め込めない。

→ **「マウスで図を組み、裏側でMermaid記法を出す」ハイブリッドエディタ**を作る。

## 2. 目的・ゴール
| 区分 | 内容 |
| --- | --- |
| プロダクトゴール | GUIによる作図と Mermaid 記法の双方向同期エディタを提供する |
| ユーザー価値 | 記法を覚えなくても作図でき、出力はテキスト（Markdown互換）として残せる |
| 成功条件 | 既存 `.mmd` ファイルを読み込み→GUI編集→保存 を破壊的差分なしで往復できる |

## 3. ターゲット
- 個人利用前提（自分のドキュメント整備用）。
- 想定シーン: 設計メモ・READMEの図、議事録の構成図、思考整理。

## 4. スコープ

### 4.1 対応する図種（優先順）
| 優先度 | 図種 | 理由 |
| --- | --- | --- |
| P1 | flowchart (graph TD/LR) | 最頻出。GUIモデルとの相性も良い |
| P2 | sequence diagram | 用途が明確で構造が単純 |
| P3 | state diagram / class diagram | 構造が grid で表しやすい |
| 後回し | ER, gantt, mindmap, journey, pie | ユースケースが限定的 |

### 4.2 対応する操作
- ノード追加・削除・移動・リサイズ
- ノード形状変更（rect, round, stadium, diamond, circle 等 Mermaid に対応するもの）
- エッジ追加・削除・付け替え、矢印種別変更（実線/点線、矢頭の有無）
- ラベル編集（ノード／エッジ）
- サブグラフ（subgraph）作成・解除
- 整列・グリッドスナップ・自動レイアウト（dagre 想定）
- アンドゥ／リドゥ

### 4.3 スコープ外
- 共同編集、認証、課金、権利処理
- Mermaid 全図種の網羅
- 高度な作図（画像貼り付け、自由曲線、図形塗り分けの細部）

## 5. 機能要件

### 5.1 双方向同期（中核）
- **GUI → Text**: 図形操作のたびに Mermaid 記法を再生成し、テキストペインに即時反映。
- **Text → GUI**: テキスト編集（保存 or デバウンス）でASTを更新し、キャンバスに反映。
- **競合制御**: テキスト編集中はGUI側の自動上書きを抑止。再パース失敗時はエラーをペインに表示し、最後に成功した状態を維持。
- **冪等性**: GUI→Text→GUI のラウンドトリップで意味的同一性を保つ（順序や空白の差は許容）。

### 5.2 GUI 編集
- 左: 図形パレット（種別ごと）
- 中央: キャンバス（パン／ズーム／スナップ）
- 右: プロパティパネル（選択要素のラベル・形状・スタイル）
- 下: Mermaid テキストペイン（読み取り＋編集トグル）

### 5.3 入出力
- インポート: `.mmd`、`.md`内 `mermaid` フェンスブロック
- エクスポート: `.mmd`、SVG、PNG、クリップボードコピー
- 自動保存（ローカル）／手動保存

### 5.4 ファイル管理
- 直近開いたファイル履歴
- 単一ファイル単位の編集（ワークスペース概念は持たない）

## 6. 非機能要件
- **応答性**: 100ノード規模で操作遅延を体感しない（< 100ms）。
- **動作環境**: Windows を最低保証。可能ならクロスプラットフォーム。
- **オフライン**: ネット接続なしで完結。
- **可搬性**: 出力 `.mmd` は Mermaid 公式レンダラでそのまま描画できる。

## 7. 技術選択肢（要検討）
| レイヤ | 候補 | 着眼点 |
| --- | --- | --- |
| 描画基盤 | React Flow / Cytoscape.js / mxgraph(draw.io 系) | ノード＋エッジモデル、カスタム形状、自動レイアウトとの相性 |
| Mermaid 連携 | `mermaid` 公式パッケージ + 独自パーサで AST 抽出 | 公式に AST API は限定的なため、簡易パーサの自作が現実的 |
| 自動レイアウト | dagre / elkjs | flowchart と相性が良い |
| 配布 | Web（PWA）／ Electron ／ Tauri | 個人利用前提なら Web or Tauri が軽い |

## 8. リスク・論点
- **表現力ギャップ**: Mermaid の一部記法（クラス定義、リンク種別の組合せ等）はGUIで自然に表現しづらい。→ 「未対応構文は raw として保持し、編集不可で温存」する設計が必要。
- **双方向同期の崩壊**: テキスト編集途中の不正状態をGUIに反映するとUXが壊れる。→ パース成功時のみ反映、失敗時は差分維持。
- **レイアウト座標の永続化**: Mermaid 自体は座標を持たない。手動配置を残すならサイドカーJSON or コメントに座標を埋め込む方式の選択が必要。

## 9. オープンクエスチョン
- 配布形態は Web / デスクトップどちらを主とするか
- 「自動レイアウト中心」か「手動配置中心」か（両立する場合のデフォルト挙動）
- 既存ツール（draw.io の Mermaid エクスポート、Mermaid Live Editor）との差別化軸をどこに置くか

## 10. マイルストーン案
1. **PoC**: flowchart のみ、GUI→Text 一方向、自動レイアウト固定
2. **MVP**: flowchart で双方向同期、保存／読込、SVGエクスポート
3. **拡張**: sequence / state 対応、手動配置の永続化、`.md`埋め込み対応

## 11. 既存OSS／類似サービス調査

### 11.1 比較表

| プロジェクト | 形態 | GUI操作 | 双方向同期 | 対応図種 | 技術スタック |
| --- | --- | --- | --- | --- | --- |
| [mermaid-live-editor](https://github.com/mermaid-js/mermaid-live-editor)（公式OSS） | Web | × テキストのみ | N/A | Mermaid全種 | SvelteKit |
| [Mermaid Chart](https://www.mermaidchart.com)（公式SaaS, 非OSS） | Web | ○ Whiteboard型GUI | ○ 実装済 | flowchart, class（拡張中） | 非公開 |
| [Mermaid Flow](https://www.mermaidflow.app/)（商用, 非OSS） | Web | ○ Drag & Drop | 不明 | flowchart のみ | 非公開 |
| [saketkattu/mermaid-visual-editor](https://github.com/saketkattu/mermaid-visual-editor)（OSS） | Web/ローカル | ○ Drag & Drop（14形状対応） | × **GUI→Text の一方向のみ**（双方向は長期課題） | flowchart のみ | Next.js + **React Flow** + **Zustand** + **Dagre** + Tailwind |
| draw.io（OSS） | Web/Desktop | ○ | △ Mermaid記法のインポート／エクスポート | 多数（汎用） | 独自 mxgraph |

### 11.2 参考にする実装ポイント
- **技術スタックの実証**: saketkattu の `React Flow + Dagre + Zustand` 構成は本件の要求と一致しており、PoC の出発点として採用候補にする。
- **ノード形状マッピング**: saketkattu は Mermaid のノード形状14種類を GUI 形状に対応付け済。そのマッピング表は再利用できる。
- **座標の永続化方法**: saketkattu は `.json` 形式のサイドカー保存を採用し、Mermaid 本体のテキスト性を保っている。本件でも同方針を有力候補とする（Mermaid側にコメント埋め込みする案より破壊が少ない）。
- **Whiteboard UX**: Mermaid Chart は「自動レイアウトと手動配置を切替可能」「テキストと図を双方向同期」を実現済。商用ベンチマークとして UX 上の到達点を意識する。

### 11.3 差別化の軸
- **OSSで双方向同期を実装する**: saketkattu は一方向止まり、双方向は商用 Mermaid Chart のみ → **OSS×双方向**は明確な空白領域。本プロダクトの中核価値をここに置く。
- **ローカル／オフライン完結**: 個人利用前提で、アカウント・課金・クラウド保存を持たない軽さを担保。
- **flowchart 以外への早期対応**: 既存OSSの GUI 編集は flowchart 偏重。sequence / state の対応が早期にできれば差別化要素になる。

### 11.4 採用しない方針（既存サービスで満たされている領域）
- 共同編集／URL共有（mermaid.live が既存）
- クラウドストレージ・チーム機能（スコープ外）
- Mermaid 全図種の網羅（公式 mermaid.live が担う領域）
