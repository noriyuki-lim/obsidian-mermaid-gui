## 横断的な前提・決定事項

実装前に固めるべき設計判断。複数項目に波及するためここに集約する。

- **【要決定 A】Available / Under Construction の判定基準**
  `supportsGui` フラグ（`src/core/templates.ts`）は「専用エディタの有無」しか表さず、block-beta / xychart のようにフォーム入力のみの図種も `true`。メモの「ドラッグ&ドロップ等のグラフィカル操作ができる図種」とは粒度が違う。
  → 推奨: テンプレートに `editorStage: "available" | "wip"` を追加し、グラフィカル編集（canvas / 操作可能プレビュー）が成立している図種だけ `available`。`supportsGui` は流用しない。

- **【要決定 B】Undo / Redo の共通実装層**
  現状、flowchart は **Toolbar に Undo / Redo ボタンを既に持つ**（`src/ui/toolbar/Toolbar.tsx`、store の `past/future`・`HISTORY_LIMIT=100`・`undo()/redo()`）。非 flowchart エディタは各々 `useState<XxxIR>` でローカル管理し、履歴もボタンも無い。`EditorShell` のツールバーも Save/Cancel + `toolbarExtras` のみ。
  → やること: (1) 非 flowchart に履歴機構を与える、(2) Undo/Redo/Export ボタンを共通バーに出す。
  → 推奨: `EditorShell` が **Mermaid ソース文字列ベースの履歴スタック**を持ち、Undo/Redo 時に `onSourceEdit(prevSource)` を呼んで各エディタの IR を再構築する。ボタンは flowchart `Toolbar` と重複実装にしないよう `<EditorActions>`（Undo/Redo/Export SVG）として切り出して両者で共有。flowchart の履歴は既存の store 実装を維持（ソース文字列履歴ではなく store 履歴を使う例外）。

- **【要決定 C】Export SVG の共通化**
  現状、Export ボタンを配線しているのは flowchart の `Toolbar` だけ（`onExportSvg`）。ただし実体 `src/obsidian/svgExport.ts` の `exportSvgToVault(app, notePath, mermaidSource)` は **任意の Mermaid ソースを `mermaid.render` で再描画して保存する図種非依存関数**。`EditorShell` にボタンが無いだけ。
  → 推奨: 共有 `<EditorActions>`（要決定 B）に Export ボタンを置き、`exportSvgToVault(app, notePath, currentSource)` を呼ぶだけ。プレビューの SVG を取り出す必要はない。
  → 留意: Mermaid 非対応図種（radar-beta / venn-beta、後述 kanban）は `mermaid.render` が失敗し Notice が出るだけなので、これらはボタンを無効化する。
  → 留意: gantt / quadrant は独自の操作可能 SVG プレビューを持つため Mermaid 再描画とは見た目が異なる。「Mermaid 版を出すか／独自プレビュー SVG を出すか」は別途決める。

## flowchart エディタ

現状: ノード新規追加時の id は `n1, n2…`（`store-factory.ts` の `newNodeId`）。一方デフォルトテンプレートは `A[Start] --> B[Step] --> C[End]` と `A/B/C`。

- デフォルトサンプルのノード id を `n1, n2, n3` に寄せる（SHAPES ドラッグ時の採番と統一）。
  → 影響: `tests/core/templates.test.ts` 等テンプレート参照テスト、`docs` のサンプル表記も追従更新。
- Subgraph を四辺ドラッグでリサイズ可能にする。
  → 現状: store の `resizeSubgraph(id, frame)` と `subgraphFrames` は**実装済み**。一方 `NodeResizer` はソース全体で未使用（grep 0 件）、`SubgraphNode` はヘッダのドラッグ移動のみ。
  → やること: `SubgraphNode` に React Flow の `NodeResizer` を付け、`onResize` を store の `resizeSubgraph` に配線するだけ。バックエンドは揃っている。
  → 留意: 座標と同じくサイズもセッション内のみ保持（ファイルには書き出さない／AGENTS の標準 Mermaid 準拠方針）。
- **Subgraph への接続**（subgraph を端点とするエッジ）。
  → 現状: 接続のラウンドトリップは **既に全層で実装済み**。`SubgraphNode` は 8 ハンドル（T/B/L/R × source/target）を持ち（`SubgraphNode.tsx`）、generator は subgraph id 端点のエッジを出力（`generator.ts:128-133`）、parser も round-trip する（`parser.ts:405-413`、adapter が `:sg:` prefix で経路付け）。ソースに `A --> sg_1` と書けば動く。
  → 唯一の不具合: GUI ドラッグでエッジを作る `addEdge` が端点を `cur.nodes` のみに限定しており（`store-factory.ts:404`）、subgraph id を端点にすると黙って drop する。これが「接続が機能していない」の正体。
  → やること: `addEdge` のガードを subgraph id も許可するよう拡張（`|| cur.subgraphs.some(s => s.id === source/target)`）。併せて、ドラッグ生成したエッジを adapter が subgraph 端点として描画できるか実機で確認。

## gantt チャートエディタ

### デフォルトソースコード

- 期間を **本日 〜 約3か月** のサンプルにする。
  → **【要決定 D】** templates.ts のテンプレートは静的文字列で、`tests/core/templates.test.ts` が「固定文字列が detect/parse を通る」ことを検証している。動的な「本日」を入れるには pick 時にテンプレートを生成する仕組みが要る。
  → 推奨: gantt のみテンプレートを `() => string` の生成関数に切り替え、本日日付を注入。テスト不変条件は「生成結果が `detectDiagramKind===gantt` かつ `parse().ok`」へ緩和。
- 各タスクに id を設定した状態にする。
- 各タスクの開始を `after <直前タスクid>` で連結（先頭タスクのみ実日付）。
- マイルストーンを 1 つ含める（後述「プレビュー」のマイルストーン表示と対応）。

### プレビュー画面

現状: 自作 SVG プレビュー（`GanttInteractivePreview`、`ROW_HEIGHT=38` / `BAR_HEIGHT=16`）。バーのドラッグ移動・左右端リサイズ・ダブルクリックでラベル編集に対応。IR には `dateFormat` のみで `axisFormat` 概念は無い。左側にタスク名を表示している。

- `axisFormat` をプルダウン（日付 / 週）＋手入力併用にする。デフォルト: 日付=`%m/%d`、週=`%W`。
  → IR に `axisFormat?: string` を追加し、generator が `axisFormat` 行を出力。
- 曜日 `%a` をチェックボックスで有効/無効。有効時は `axisFormat` に曜日要素を合成（例 `%m/%d(%a)`）。
- 左側表示を **タスク名 → セクション名** に変更（Mermaid 標準ガントの見た目に寄せる）。バー左のタスク名ラベルは廃止し、行をセクション単位でグルーピングしてセクション名を左に表示。
- **左端／右端の日付ラインをドラッグして表示期間を伸縮**。
  → **【要決定 D2】** Mermaid gantt に「表示範囲」ディレクティブは無い。これはソースに影響しない **プレビュー専用のズーム/ビューポート**（セッション内のみ、保存対象外）と解釈するのが妥当。データ範囲を変える操作ではない旨を確定する。
- **バー間ドラッグで依存線を張り `after id` を自動設定**。
  → タスク A のバーから B のバーへドラッグすると、B の開始が `after <A.id>` になる（両者に id 必須。無ければ自動採番）。
  → 依存線はガント GUI 独自の表現で、Mermaid 自体は描画しない旨に留意。
  → 線を選択し Delete キーで削除（= B の `after` を解除）。
- プレビュー上でバー選択 → Delete でタスク削除。

### テーブル欄

現状: type/label/id/status/start/end 列のテキスト入力（`mge-gantt-cell-input`）。status はカンマ区切り文字列。Tab ベースのセル移動あり。

- 日付セルはピッカー選択と `5d`/`7d` 等の期間入力を両立。
  → 操作感: `Alt + ↑/↓` でピッカー起動 → `Tab/Shift+Tab` で「日付指定 / 期間指定」を切替 → `↑/↓` で日付選択または期間を 1 日ずつ増減。
- `Tab/Shift+Tab` もしくは `←/→` で横移動、`↑/↓` で縦移動。
- `F2` でナビゲーションモード ⇔ セル編集モードを切替（Excel ライク）。セル編集モードのときに `←/→` を押したらあくまでテキスト内での移動を行う（ナビゲーションしない）。
- セル選択時に行全体の色が変わる挙動は不要（行ハイライトを外し、セル単体のフォーカス表現に）。
- STATUS をプルダウン選択式にする（`done` / `active` / `crit` / `milestone`）。`Alt + ↑/↓` で選択。
  → 現状: IR は `modifiers: GanttTaskStatus[]`（`src/core/gantt/ir-types.ts`）で複数 status を配列保持し、カンマ区切りで出力（例 `:crit, active`）。`milestone` も status 値の 1 つ。
  → **【要決定 E】** プルダウンを単一選択にするか複数選択（配列のまま）を残すか確定する。推奨: 単一選択を基本にしつつ `crit` だけ併用可（Mermaid 実態に合わせる）。

## xychart

現状: フォーム入力（title テキスト / orientation セレクト / x-axis・y-axis 設定 / series リスト）。表形式ではない。
IR（`src/core/xychart/ir-types.ts`）: `xAxis`（categorical / numeric / label-only）、`yAxis`、`items` に `XYSeriesItem { series: "bar"|"line"; values: number[] }`。**系列に名前フィールドは無い**（`bar [30,60,45,80]` のように無名で出力）。

- 「Excel テーブルから棒グラフ」の体験に寄せ、データ入力欄を **表形式（縦持ち）**にする。
  → レイアウト: **1 列目 = x 軸カテゴリ（行方向に並ぶ）**、2 列目以降 = 各系列の値。列ヘッダは系列名ではなく **bar / line のトグル**（IR に系列名が無いため。「系列1/2」は表示専用ラベル）。右端「＋」で系列追加、最下行「＋」で x カテゴリ追加。
  → 例（デフォルトテンプレート相当）:

    | x-axis | 系列1〔bar〕 | 系列2〔line〕 |
    |---|---|---|
    | Jan | 30 | 30 |
    | Feb | 60 | 60 |
    | Mar | 45 | 45 |
    | Apr | 80 | 80 |

  → これは次のソースと round-trip する:

    ```
    xychart-beta
      x-axis [Jan, Feb, Mar, Apr]
      bar [30, 60, 45, 80]
      line [30, 60, 45, 80]
    ```

  → Excel 貼り付け: 1 列目を x カテゴリ、残り列を系列値として取り込む。系列種別は取り込み後にヘッダのトグルで指定（デフォルト bar）。
  → x 軸が numeric / label-only のときはカテゴリ列がそれぞれ数値 / ラベルになる点を整理（categorical を主動線とする）。
- title / orientation / y-axis（title・min・max、auto 可）は表の上部にフィールドとして残す。
