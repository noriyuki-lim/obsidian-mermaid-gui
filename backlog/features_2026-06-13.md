# 改善メモ（図種別・ステータス整理）

## 未対応

### 1. 横断課題 / 共通機能
- エディタ上で、ホバーしたときの説明ポップアップは不要（ポップアップが2つ重なってしまっている。Obsidianが出しているポップアップと重複しているのか不明なので調査は必要）

### 2. flowchart エディタ
- Editor開いたときにエッジがノードに対して微妙に左にずれて見える。Auto-layout押すと治るのだが、再度エディターを開きなおすとまたずれてしまっている。右に少し補正するとかいう暫定的な対応では根本解決に至らないため、原因究明したうえで、何か無駄な処理が入っているのであれば抜く、あるいは追加処理が必要なら追加するなり最適な方式をとって、自動整列された状態で表示してほしい。
- Sort source by Canvas の挙動が安定しない。並び順判定のロジックを改善してほしい。

### 3. gantt チャートエディタ
- 特になし

### 4. kanban エディタ
- カラムも掴んで左右並び替えできるようにする。もちろんmermaidソースにも順序が反映されるような実装とすること。

### 5. xychart エディタ
- バードラッグでの値変更は、バーの上部のみドラッグで有効とする。バーの中間付近をドラッグしても何も起こらない。ダブルクリックで値編集は残す。
- バードラッグでの値変更は1単位にする。小数点以下はスキップする。
- lineチャートもノード部分のドラッグで上下に1単位で値変更できるようにする。
- lineチャートのノードにホバーするとダブルクリック可能なことが分かりやすいようにカーソルデザインを変えたい。

### 6. block-beta エディタ
- 特になし

## 対応済

### 1. 横断課題 / 共通機能
#### パネルレイアウトの永続化
- **プレビュー・ソースパネルの幅/高さを図種ごとに記憶**
  - **現状**: エディタを閉じて再度開くと、プレビュー幅・高さの比率は毎回共通デフォルト値に戻っていた。
  - **変更**: `src/ui/layoutPrefs.ts` を新設し、`EditorShell` の side ratio / preview ratio をドラッグ確定時に `localStorage` へ保存。保存キーは `diagramKind` で名前空間化し、図種ごとに独立して幅・高さの好みを保持する（ある図種で広げても他の図種には影響しない）。flowchart 専用 `TextPane` のソースペイン高さも同様に永続化。

#### レイアウトの統一
- **操作可能プレビュー主体エディタの `layout="stacked"` 統一**
  - **現状**: gantt のみ `layout="stacked"`。kanban / xychart / block / quadrant は既定 `side`。
  - **変更**: kanban、xychart、block などを `stacked` 化。操作可能プレビューを全幅表示し、フォームやテーブルを下部に配置。
  - **留意/影響**: stacked の preview 行の高さや、下段 of 縦幅確保などの調整を実施。

#### 前提・決定事項
- **Available / Under Construction の判定基準**
  - **現状**: `supportsGui` フラグが「専用エディタの有無」しか表さず、グラフィカル編集非対応でも true になっていた。
  - **変更**: テンプレートに `editorStage: "available" | "wip"` を追加し、グラフィカル編集が可能な図種のみ `available` に。
- **Undo / Redo の共通実装層**
  - **現状**: flowchart のみ store に Undo/Redo 機能があり、他は履歴もボタンもなかった。
  - **変更**: `EditorShell` が Mermaid ソース文字列ベース of 履歴スタック（past/future）を持ち、Undo/Redo 時に `onSourceEdit(prevSource)` を呼ぶように共通化。
- **Export SVG の共通化**
  - **現状**: flowchart のみに Export ボタンがあった。
  - **変更**: 共通コンポーネント `<EditorActions>` に Export ボタンを配置し、`exportSvgToVault` を呼び出すように共通化。

#### 新規作成パネル（DiagramKindPicker）
- **Available / Under Construction のセクション分け廃止**
  - **現状**: Available / Under Construction セクションにグループ分けして表示していた。
  - **変更**: グルーピングを廃止しフラットな一覧に戻した（Favorites ★ のみ維持）。判定に使っていた `editorStage` フィールドも未使用になったため削除。旧 Available（グラフィカルな直接操作エディタを持つ図種: flowchart / quadrantChart / gantt / block-beta / kanban）を `DIAGRAM_TEMPLATES` 配列の先頭にまとめ、xychart-beta は kanban の直後に配置。
- **タイルのドラッグ並べ替え**
  - **現状**: 表示順は `DIAGRAM_TEMPLATES` 配列順（および Favorites ピン留め）でしか変えられなかった。
  - **変更**: タイル左上にグリップハンドルを追加し、ドラッグで自由に並べ替え可能に（ポインタイベントベースで掴んでいる間も動的に再描画）。並び順は `mge-kind-order` として localStorage に永続化し、新規/削除されたテンプレートは自動でマージする。
- **リード文の文体・内容修正**
  - **現状**: 「図の種別を選択。テンプレートが読み込まれ、続きを GUI で編集できる。☆ で上部にピンできる。」（常体、GUI編集前提の書きぶり）。
  - **変更**: ですます調に統一し、GUI直接編集またはフォーム中心編集のどちらもあることを明記。
- **チャートタイルの説明見切れ防止**
  - **現状**: タイル内の説明テキストが見切れていた。
  - **変更**: タイル縦幅を確保し、動的に折り返すよう CSS を調整。
- **TEMPLATE SOURCE 欄の縦幅確保**
  - **現状**: テンプレートソース欄の縦幅が極狭だった。
  - **変更**: `aside` を `flex: 1` で縦に占有させ、ポップアップ下端まで伸ばした。
- **お気に入りピン留め機能**
  - **現状**: なし。
  - **変更**: タイル内の星アイコンで図種をピン留めし、Available セクションの先頭に「★ お気に入り」として固定表示（localStorage 保存）。

#### エディタ共通
- **上部バーへのアクション配置**
  - **現状**: 各エディタ固有のヘッダを使用していた。
  - **変更**: 上部バーに Undo / Redo / Export SVG ボタンを共通配置。
- **flowchart 固有ボタンの移動**
  - **現状**: 共通バーに flowchart 固有の Direction / Subgraph / Auto-layout ボタンが同居していた。
  - **変更**: Direction / Subgraph は左パネル（Palette）上部へ移動し、Editor edge / Auto-layout はキャンバス左上へ配置。共通バーには共通アクションのみ配置。
- **Export SVG ファイル名のタイムスタンプ**
  - **現状**: UTC 時間で出力。
  - **変更**: 東京時間（JST, Asia/Tokyo）のタイムスタンプへ変換。
- **Modal の最大化 / 復元**
  - **現状**: toolbar ドラッグによる移動と四隅リサイズのみ対応していた。
  - **変更**: toolbar の空白部分をダブルクリックすると最大化 / 復元を切り替える。最大化 / 復元には短い矩形 transition を入れる。最大化状態から toolbar をドラッグした場合は、掴んだ位置がマウス直下に来るよう復元矩形を調整して自由配置へ移行する。初期の中央配置状態では四隅リサイズ時に中心位置を保って拡縮し、toolbar ドラッグ後は自由配置として掴んだ角だけを動かす。

### 2. flowchart エディタ
#### キャンバスコントロール
- **Direction / Subgraph / Editor edge / Auto-layout コントロールの配置**
  - **現状**: 左パネル（Palette）上部に配置されていた。
  - **変更**: Mermaid ソースに残る Direction / Subgraph は Palette の Shapes 上部に配置。エディタ上の見え方を制御する Editor edge（Bezier / Smooth step）と Auto-layout はキャンバス左上の `FlowchartCanvasControls` に縦並びで配置。Editor edge の選択は `localStorage` に保持し、Mermaid ソースには書き込まない。
- **Modal リサイズ時のキャンバス中心維持**
  - **現状**: ウィンドウサイズ変更時に React Flow の viewport が左上固定に見えていた。
  - **変更**: キャンバスコンテナのリサイズ時は、リサイズ前の画面中心にあった flow 座標をリサイズ後も中心に保つ。Modal 最大化 / 復元の transition 中は補正を保留し、transition 終了後に1回だけ補正する。

#### ノード・サブグラフ
- **ノード ID の統一**
  - **現状**: 新規追加 ID は `n1, n2` だが、テンプレートは `A, B, C` であった。
  - **変更**: デフォルトサンプルのノード ID を `n1, n2, n3` に統一。
- **Subgraph の四辺リサイズ**
  - **現状**: store には実装があったが、UI で NodeResizer が使われていなかった。
  - **変更**: `SubgraphNode` に `NodeResizer` を導入し、ドラッグでのリサイズに対応（サイズはセッション内のみ保持）。
- **Subgraph 内 Direction**
  - **現状**: parser / generator は `subgraph` 内の `direction` を扱えたが、GUI から編集できず、キャンバス上の配置にも反映されなかった。
  - **変更**: Subgraph 選択時の右ペインに Direction を追加し、`(inherit)` / TD / LR / BT / RL を選択可能にする。変更時と Auto-layout 時は subgraph 内を先に局所レイアウトし、外側の Dagre では subgraph を表示 frame と同じ box node として扱う。Mermaid ソースにも `direction ...` を出力する。
- **Subgraph への接続**
  - **現状**: parser/generator などのバックエンドは subgraph 端点のエッジに対応していたが、GUI 上の `addEdge` ドラッグ時に subgraph id の接続を drop していた。
  - **変更**: `addEdge` で subgraph id を許容するように拡張し、GUI での接続を可能に。
- **Connected edges 一覧**
  - **現状**: 同じノードに繋がっている edge が複数あるとき、キャンバス上では重なった edge の一部しか掴みにくかった。
  - **変更**: ノード選択時の右ペインに connected edges 一覧を表示し、edge の選択・削除・接続先変更を一覧から操作できるようにした。選択済み edge はキャンバス上でも前面化し、reconnect でも選択済み edge が1本かつヒットした edge と固定端点を共有する場合は、その選択済み edge を操作対象にする。
- **Sort source by canvas**
  - **現状**: 同じ階層レベルにあるノードのキャンバス上の並び順が Mermaid コードの出力順に反映されなかった。
  - **変更**: `Sort source by canvas` で現在座標に基づいて node / subgraph / edge の出力順を整列できるようにした。

### 3. gantt チャートエディタ
#### デフォルトソースコード
- **動的テンプレート**
  - **現状**: 静的な固定文字列テンプレートだった。
  - **変更**: 今日〜約3ヶ月の期間を動的に生成する関数テンプレート化。
- **タスク・マイルストーン設定**
  - **現状**: タスク ID や `after` 接続、マイルストーンのないシンプルな状態。
  - **変更**: 各タスクに ID を設定し、`after <id>` で依存関係を連結。マイルストーンを 1 つ含めた構成に。

#### プレビュー画面
- **全幅化とレイアウト調整**
  - **現状**: viewBox 固定のため全幅でも横に余白が出ていた。
  - **変更**: SVG をコンテナ幅に追従させ、座標変換も動的幅に連動。
- **設定オプションのプレビュー側配置**
  - **現状**: axisFormat / 曜日などの設定 UI がテーブル側にあった。
  - **変更**: プレビュー上部のツールバー（`.mge-gantt-preview-tools`）へ移設。
- **設定変更のプレビュー即時反映**
  - **現状**: プレビューの目盛り描画が `axisFormat` などの設定を読んでいなかった。
  - **変更**: `axisFormat`（日付/週）、曜日トグル（有効時は `%a` を合成）の設定を反映して日付目盛りラベルを再描画。
- **プレビュー上での操作**
  - **並べ替え**: 左端に ID/ドラッグハンドル列を設け、掴んでタスク行を並べ替え（同一セクション内並べ替え・セクション跨ぎでの所属セクション変更）。
  - **表示期間の伸縮**: プレビュー左右端の日付ラインをドラッグして、ビューポート表示範囲を伸縮（セッション内のみ）。
  - **依存線の接続** : バー間ドラッグで `after id` の依存関係を設定。Delete キーで依存線を削除。
  - **タスク削除**: バー選択中に Delete キーでタスクを削除。
- **依存関係を維持したバーリサイズ**
  - **現状**: `after` でつながっているバーの開始部分をドラッグすると依存関係が外れていた。
  - **変更**: 開始部分のドラッグ時も `after` 依存を維持し、直前依存タスクの終了部分だけを伸縮するように変更。
  - **変更**: 終了部分のドラッグ時は直後依存タスクの接続部分だけを更新し、他タスクの開始・終了地点は基本維持するように変更。

#### テーブル欄
- **日付入力の改善**
  - **現状**: 日付セルが素のテキスト入力で、ピッカーも機能していなかった。
  - **変更**: start/end セル内で `date` / `after` / `dur` を明示選択できるようにし、空欄状態からでも日付・依存・期間を入力できるようにした。
  - **変更**: `date` はネイティブ日付ピッカーをセル内ボタンとして表示し、start/end のテキスト入力自体には不要な日付ドロップダウンを持たせない形に整理。
  - **変更**: `date` 編集中は `YYYY-MM-DD` のキャレット位置に応じて、編集モード中の `↑/↓` で年・月・日を個別に増減できるようにした。
  - **変更**: `after` は `Alt + ↑/↓` で依存先 ID を選択し、`dur` は編集モード中のみ `↑/↓` で日数を増減する挙動に整理。
  - **変更**: ピッカー表示中の `Esc` はピッカーだけを閉じ、エディタ全体を閉じないようにした。
- **キーボードナビゲーションの Excel ライク化**
  - **現状**: 行全体がハイライトされ、セル単位のフォーカス操作が不十分だった。
  - **変更**: 行ハイライトを外しセル単体のフォーカスへ変更。`F2` でナビゲーションモードと編集モードを切り替え、`Tab/Shift+Tab`/矢印キーで縦横のセル移動を可能に。
- **STATUS 列のプルダウン化**
  - **現状**: modifier がカンマ区切りのテキスト入力だった。
  - **変更**: `active` / `done` / `crit` / `milestone` をプルダウン選択できるようにし、`Alt + ↑/↓` で STATUS 候補を選択可能に（crit 併用可）。
- **視覚的バグの補修**
  - **現状**: crit トグルと START 欄の境界線が消えて見えていた。
  - **変更**: 縦罫線を描画し、STATUS セル内に収まっていることを視覚的に明示。
- **行並べ替えの同期**
  - **変更**: テーブル側での並べ替えとプレビュー側の並べ替えモデル（`reorder`）を共通化。

### 4. kanban エディタ
#### プレビュー画面

  - **現状**: side レイアウトで右 4 割に表示、左に操作説明テキスト。
  - **変更**: `layout="stacked"` にし、左の説明セクションを廃止。操作説明はボード上部へ小さく表示。
- **カラムの横スクロール**
  - **現状**: 列が増えた時に side の狭い枠で潰れていた。
  - **変更**: ボードラッパで横スクロール（`overflow-x: auto; flex-wrap: nowrap`）を適用。

  - **現状**: 未実装（新規図種）。
  - **変更**: `KanbanInteractivePreview` を新設。カラム間でのカードの DnD 移動、カラム・カードの追加・削除・インライン編集をサポートする bespoke ボードを実装。

### 5. xychart エディタ
#### プレビュー画面
- **全幅化と操作可能プレビュー新設**
  - **現状**: 操作可能プレビューを持たず、静的 Mermaid プレビューを表示していた。
  - **変更**: 自作インタラクティブ SVG プレビュー（`XYChartInteractivePreview`）を導入し、`layout="stacked"` で全幅表示.
- **プレビュー上の編集**
  - **変更**: カテゴリ名（x 軸ラベル）のインライン編集、バーダブルクリックでの数値編集、バー上端ドラッグでの数値増減に対応。
  - **折れ線（line 系列）**: MVP 推奨方針に従い、値表示とダブルクリック編集のみに留め、ドラッグ対象外に。

#### テーブル欄
- **縦持ち表形式データ入力**
  - **現状**: 系列ごとのリストや個別のフォーム入力だった。
  - **変更**: Excel のような縦持ちテーブル（1 列目＝x 軸カテゴリ、2 列目以降＝系列値）を導入。ヘッダで系列種別（bar/line）をトグル選択。
- **入力フォームの配置**
  - **変更**: title / orientation / yAxis は表の上部に配置。
- **Excel からの貼り付け・キーボード操作**
  - **変更**: gantt と同様の Excel ライクなキーボード操作（F2切替、セル移動、Alt+↑/↓で系列切替や数値調整など）および TSV 形式での貼り付け機能を実装。
- **Y 軸タイトルの分離**
  - **現状**: auto チェックボックスをオンにすると Y 軸タイトル入力欄まで消えていた。
  - **変更**: auto は min/max 入力欄のみを支配し、Y 軸タイトルは常時編集可能に。

## 7. quadrantChart エディタ

### 対応済
#### レイアウト
- **プレビュー幅・高さの初期値**
  - **現状**: 全図種共通のデフォルト比率（side 42% / preview 58%）で表示され、真ん中より狭い上に mermaid ソース欄が広めだった。
  - **変更**: `EditorShell` に `defaultSideRatio` / `defaultPreviewRatio` を追加し、quadrantChart はプレビュー幅 55%・高さ比率 68% を初期値に指定（保存済みの好みがまだ無い場合のみ影響。ユーザーがドラッグ調整すればそちらが優先される）。

#### プレビュー描画
- **テキストサイズの実機合わせ**
  - **現状**: 象限ラベル・軸ラベル・ポイントラベルの文字が、実際の Mermaid レンダリングより大きすぎた。
  - **変更**: 実機スクリーンショットの比率を測定し、象限ラベル 4px→2.6px（20%追加縮小含む）、軸ラベル 3.5px→2.3px、ポイントラベル 3.5px→2.4px に調整。
- **Quadrant Labels の位置**
  - **現状**: 象限中央に表示されていた。
  - **変更**: 象限上端付近（y=4 / 54）に寄せ、実際のレンダリングに倣った配置に変更。
- **軸ラベルの配置**
  - **現状**: x軸・y軸ラベルをプロット内部（0–100 の範囲内）に描画しており、実際のレンダリング（プロット枠の外側に配置）と異なっていた。
  - **変更**: SVG の viewBox に左・下マージンを追加し、x軸ラベルはプロット下部の外側、y軸ラベルはプロット左側の外側に `rotate(-90)` で縦書き表示するよう変更。
- **ポイントの大きさ・ラベル位置**
  - **現状**: 半径 2.4（ドラッグ中 3.2）と大きく、ラベルはポイント右上に表示されていた。
  - **変更**: 半径 1.3（ドラッグ中 1.8）に縮小して実際のレンダリングに近づけ、ラベルはポイント直下に中央揃えで表示するよう変更。
- **ポイントドラッグ時のカーソルずれ修正**
